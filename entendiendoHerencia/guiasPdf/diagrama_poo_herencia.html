public virtual int GetAtaque()// hija puede redefinir

public bool EstaVivo()// _vida > 0

class Barbaro : Personaje

CLASE HIJA
Atributo propio — encapsulado
private int _furia;// no existe en Personaje
Constructor — agrega _furia
public Barbaro(string n,int v,int a,int f)
: base(n, v, a)// ← padre inicializa n,v,a
{ _furia = f; }// ← hijo inicializa _furia
override CalcularDanio() — reemplaza al padre
public override int CalcularDanio()
si _furia ≥ 3 → daño × 1.15, furia −= 3
si _furia < 3 → daño × 0.50 (cansado)
⚠ NO llama base.CalcularDanio() — reemplazo total

class Zamuray : Personaje

CLASE HIJA
Atributo propio — flag de desarme
private bool _desarmar = false;
2 overrides + método propio
public override int CalcularDanio()// activa flag
public override int GetArmadura()// siempre retorna 0
public void AplicarHabilidad(Personaje obj)// propio
si _desarmar → obj.QuitarEquipo()
⚠ AplicarHabilidad NO existe en Personaje
→ se llama con pattern matching: p is Zamuray z
🔀 virtual / override
virtual en el padre = "este método puede ser reemplazado".
override en la hija = "lo reemplazo con mi versión".
Sin ambos, C# ejecuta siempre la versión del padre.
virtual → padre propone
override → hija responde
📞 base() y base.Metodo()
base(n,v,a) en el constructor → delega al padre la inicialización de los campos heredados.
base.Metodo() en un override → ejecuta la lógica del padre antes/después de la propia.
base(n,v,a) → constructor padre
base.RecibirDanio(d) → Sacerdote
🔒 Encapsulación
Atributos private = el estado interno solo lo modifica la propia clase.
Getters public = única puerta de lectura.
Cada hija encapsula sus propios atributos extra (_furia, _desarmar).
private int _furia;
public int GetVida() => _vida;
⚡ POLIMORFISMO — dispatch dinámico (el núcleo del juego)
Tipo declarado vs tipo real
Personaje p = new Barbaro(...)
variable tipo base → objeto tipo hija
p.CalcularDanio();
→ C# ejecuta el override de Bárbaro, no la base
esto es late binding / despacho dinámico
En Batalla() — mismo código, distintos tipos
Batalla(Personaje p1, Personaje p2)
p1 puede ser Barbaro, Sacerdote o Zamuray
int d1 = p1.CalcularDanio();
p2.AplicarDanio(d1);
→ Batalla() no sabe ni necesita saber el tipo real
Pattern matching — is + cast
if (p1 is Zamuray z)
z.AplicarHabilidad(p2);
AplicarHabilidad no existe en Personaje → necesita el tipo concreto
→ is verifica el tipo real en runtime y castea
